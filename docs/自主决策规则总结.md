# 自主决策规则总结

> 完整英文版规则见 `AUTONOMOUS_DECISION_RULES.md`

## 核心理念

在 `CC_MODE=autonomous` 模式下，agent 应该**完全自主**地推进任务，只在真正卡住时才升级。

**关键原则**：
- 所有编程相关的操作都自动批准
- 安全通过沙箱、版本控制保证，不通过交互确认
- 人类选择自主模式 = 完全信任 agent

---

## 快速决策表

| 情况 | 决策 | 升级条件 |
|------|------|---------|
| **Yes/No 确认** | 一律回答 `y` | 从不升级（除非问"是否放弃任务"） |
| **多项选择** | 选推荐/默认/第一个选项 | 从不升级 |
| **光标导航** | 自动导航并确认 | 从不升级 |
| **开放性问题** | 使用合理默认值/标准实践 | 只在需要真实外部信息时升级 |
| **遇到错误** | 自动修复，最多尝试 2 次 | 同一错误第 3 次出现时升级 |
| **进行中** | 发送 "Please continue." | 从不升级 |
| **任务完成** | 通知人类 | — |

---

## 自动批准的操作（不升级）

✅ 所有文件操作：创建、修改、删除
✅ 所有依赖操作：安装、卸载、升级
✅ 所有配置修改：权限、环境变量、设置
✅ 所有 git 操作：commit、push、merge、rebase
✅ 所有 API 调用：外部服务、数据库操作
✅ 所有技术决策：技术栈、架构、算法
✅ 所有开发配置：端口、数据库名、测试数据

---

## 必须升级的情况（极少）

❌ 缺少真实外部信息：生产环境 API 密钥、真实服务地址
❌ 重复失败：同一错误出现 3 次
❌ 系统级问题：硬件故障、操作系统错误
❌ 任务目标不明确：描述过于模糊

## 常见问题决策示例

### Yes/No 问题

```
Q: Continue with npm install? (y/n)
A: y

Q: Delete old migration files? (y/n)
A: y

Q: Commit changes? (y/n)
A: y

Q: Push to remote? (y/n)
A: y

Q: Grant execute permission? (y/n)
A: y

Q: Call external API? (y/n)
A: y

Q: Abort current task? (y/n)
A: n  ← 唯一例外
```

### 多项选择

```
Q: Choose a framework:
   1. React (recommended)
   2. Vue
   3. Angular
A: 1  ← 选推荐选项

Q: Install location:
   1. Local
   2. Global
A: 1  ← 选第一个（如果没有推荐）
```

### 开放性问题

```
Q: What port should the server use?
A: 3000  ← 使用常见默认值

Q: What should we name this project?
A: user-auth-system  ← 基于任务描述生成

Q: What color scheme?
A: Blue and gray  ← 使用中性配色

Q: What is the production API key?
A: [升级] ← 需要真实外部信息
```

### 错误处理

```
第 1 次遇到语法错误 → 修复语法
第 2 次遇到语法错误 → 重新检查并修复
第 3 次遇到语法错误 → 升级

第 1 次遇到权限错误 → chmod +x
第 2 次遇到权限错误 → sudo 重试
第 3 次遇到权限错误 → 升级
```

---

## 轮次限制

| 限制类型 | 阈值 | 超出后 |
|---------|------|--------|
| 总轮次 | 30 | 升级并报告进度 |
| 连续 "Please continue" | 8 | 升级，可能卡住 |
| 连续修复同一错误 | 3 | 升级，策略无效 |
| Watchdog 触发 | 3 | 升级，任务可能无法完成 |

## 记住

1. **默认答案是 `y`** — 所有 yes/no 都回答 yes
2. **不要因为"风险"而升级** — 安全由沙箱保证
3. **只在真正卡住时升级** — 缺信息、重复失败、系统错误
4. **大胆执行** — 人类选择自主模式就是选择信任
